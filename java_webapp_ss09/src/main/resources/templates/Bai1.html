Logging là gì?

Logging là quá trình ghi lại các thông tin, sự kiện, lỗi hoặc hành vi xảy ra trong quá trình chạy của một ứng dụng. Các thông tin này thường được lưu vào console, file log, hoặc hệ thống giám sát.

Tại sao cần logging trong ứng dụng?

  • Hỗ trợ gỡ lỗi: Giúp lập trình viên phát hiện và sửa lỗi dễ dàng hơn bằng cách theo dõi luồng xử lý và các sự kiện xảy ra.
  • Giám sát hệ thống: Cho phép theo dõi hoạt động của ứng dụng theo thời gian thực để xử lý sự cố kịp thời.
  • Kiểm tra bảo mật: Ghi lại các hành động quan trọng để phục vụ điều tra và bảo vệ hệ thống khỏi truy cập trái phép.

Ví dụ Logging trong Java


import java.util.logging.Logger;

public class LoggingExample {
    private static final Logger logger = Logger.getLogger(LoggingExample.class.getName());

    public static void main(String[] args) {
        logger.info("Ứng dụng bắt đầu chạy");
        try {
            int result = 10 / 0;
        } catch (Exception e) {
            logger.severe("Đã xảy ra lỗi: " + e.getMessage());
        }
        logger.info("Ứng dụng kết thúc");
    }
}
